# 設定範例：工作階段恢復 (Session Resume)

示範使用 Copilot SDK 進行工作階段持久化與恢復。這驗證了已銷毀的工作階段可以透過其 ID 恢復，並保留完整的對話歷程記錄。

## 每個範例的操作內容

1. 使用 `availableTools: []` 和模型 `gpt-4.1` 建立一個工作階段
2. 發送：_"記住這點：秘密單字是 PINEAPPLE。"_
3. 獲取工作階段 ID 並銷毀該工作階段
4. 使用相同的工作階段 ID 恢復工作階段
5. 發送：_"我告訴過你的秘密單字是什麼？"_
6. 列印回應 —— 回應應提到 **PINEAPPLE**

## 設定

| 選項 | 值 | 效果 |
|--------|-------|--------|
| `availableTools` | `[]` (空陣列) | 保持工作階段簡單，不使用工具 |
| `model` | `"gpt-4.1"` | 在初始和恢復的工作階段中均使用 GPT-4.1 |

## 執行

```bash
./verify.sh
```

需要 `copilot` 二進位檔案 (自動偵測或設定 `COPILOT_CLI_PATH`) 和 `GITHUB_TOKEN`。

# 設定範例：並行工作階段 (Concurrent Sessions)

示範在 **同一個用戶端上建立多個工作階段**，並配置不同的設定，驗證每個工作階段都能維護其各自隔離的狀態。

## 測試內容

1. **工作階段隔離** — 在同一個用戶端上建立的兩個工作階段會接收不同的系統提示，並根據各自的角色 (persona) 而非對方的角色進行回應。
2. **並行操作** — 兩個工作階段可以並行使用而互不干擾。

## 每個範例的操作內容

1. 建立一個用戶端，然後同時開啟兩個工作階段：
   - **工作階段 1** — 系統提示：_"你是一個海盜。總是要說 Arrr!"_
   - **工作階段 2** — 系統提示：_"你是一個機器人。總是要說 BEEP BOOP!"_
2. 向兩個工作階段發送相同的問題 (_"法國的首都是哪裡？"_)
3. 列印兩個回應並標註標籤 (`Session 1 (海盜):` 和 `Session 2 (機器人):`)
4. 銷毀兩個工作階段

## 設定

| 選項 | 工作階段 1 | 工作階段 2 |
|--------|-----------|-----------|
| `systemMessage.mode` | `"replace"` | `"replace"` |
| `systemMessage.content` | 海盜角色 | 機器人角色 |
| `availableTools` | `[]` | `[]` |

## 執行

```bash
./verify.sh
```

需要 `copilot` 二進位檔案 (自動偵測或設定 `COPILOT_CLI_PATH`) 和 `GITHUB_TOKEN`。

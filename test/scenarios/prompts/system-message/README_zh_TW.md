# 設定範例：系統訊息

示範如何使用 `replace` 模式設定 Copilot SDK 的 **系統訊息 (system message)**。這驗證了自訂系統提示可以完全取代預設系統提示，從而改變代理的個性與回應風格。

## 附加 (Append) vs 取代 (Replace) 模式

| 模式 | 行為 |
|------|----------|
| `"append"` | 將您的內容新增到預設系統提示 **之後**。代理保留其基本個性以及您的補充內容。 |
| `"replace"` | 使用您的內容 **取代** 整個預設系統提示。代理的個性完全由您的提示定義。 |

## 每個範例的操作內容

1. 使用海盜個性提示，以 `replace` 模式建立具有 `systemMessage` 的工作階段
2. 發送：_"法國的首都是哪裡？"_
3. 列印回應 —— 回應應該是海盜口吻 (包含 "Arrr!"、航海術語等)

## 設定

| 選項 | 值 | 效果 |
|--------|-------|--------|
| `systemMessage.mode` | `"replace"` | 完全取代預設系統提示 |
| `systemMessage.content` | 海盜個性提示 | 指示代理始終以海盜口吻回應 |
| `availableTools` | `[]` (空陣列) | 無工具 —— 將測試重點放在系統訊息行為上 |

## 執行

```bash
./verify.sh
```

需要 `copilot` 二進位檔案 (自動偵測或設定 `COPILOT_CLI_PATH`) 和 `GITHUB_TOKEN`。

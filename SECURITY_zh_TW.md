# 安全性 (Security)

GitHub 嚴肅對待我們軟體產品和服務的安全性，包括透過我們的 GitHub 組織 (例如 [GitHub](https://github.com/GitHub)) 管理的所有開源程式碼儲存庫。

儘管 [開源儲存庫超出了我們錯誤懸賞計畫的範圍](https://bounty.github.com/index.html#scope)，因此不符合懸賞獎勵資格，但我們將確保您的發現被轉遞給適當的維護者進行補救。

## 回報安全性問題

如果您認為在任何 GitHub 擁有的儲存庫中發現了安全性弱點，請透過協調披露 (coordinated disclosure) 向我們回報。

**請勿透過公開的 GitHub Issue、討論或提取請求 (Pull Request) 回報安全性弱點。**

請改為發送電子郵件至 opensource-security[@]github.com。

請盡可能包含下列列出的資訊，以協助我們更好地了解並解決此問題：

- 問題類型 (例如：緩衝區溢位、SQL 注入或跨網站指令碼)
- 與問題表現相關的原始檔案完整路徑
- 受影響原始碼的位置 (標籤/分支/提交或直接 URL)
- 重現問題所需的任何特殊設定
- 重現問題的逐步說明
- 概念驗證 (Proof-of-concept) 或攻擊程式碼 (如果可能)
- 問題的影響，包括攻擊者可能如何利用該問題

這些資訊將協助我們更快速地對您的報告進行分類。
